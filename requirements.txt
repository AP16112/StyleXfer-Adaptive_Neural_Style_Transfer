#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# Flask :-
# A lightweight Python web framework. we are using it to build the web interface (Flask, render_template, request, etc.) for uploading images and running style transfer.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# Flask_Bootstrap :-
# Integrates Bootstrap (CSS/JS framework) with Flask templates. 
# This gives our web app a polished, responsive UI without writing raw CSS.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# flask_wtf :- 
# Adds support for forms in Flask using WTForms. we are using it for file upload forms (FlaskForm, FileField, SubmitField, etc.).
# Extension that integrates WTForms with Flask. It simplifies form handling (upload fields, validation, CSRF protection). You’re using it for FlaskForm and FileField

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# Werkzeug :-
# A core library Flask depends on for routing, request handling, and utilities like secure_filename.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# WTForms :-
# Provides form field classes and validators (FileField, FloatField, InputRequired, etc.) used in our upload and parameter forms.
# A form library. Provides the actual form field classes (FileField, FloatField, SubmitField) and validators (InputRequired). Flask-WTF builds on this.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# Pillow :-
# The Python Imaging Library fork. 
# We are using it to load, save, and manipulate images (PIL.Image.open, resizing, etc.). 
# Essential for handling input/output images in our project.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# torch :-
# The main deep learning framework. Provides tensors, autograd, optimizers, and neural network modules. Our AdaIN implementation (VGGEncoder, Decoder) is built on this.

# torchvision :-
# Companion library for PyTorch. 
# Provides image transforms (torchvision.transforms), datasets, and utilities like save_image.
# Crucial for preprocessing and saving styled outputs.

#  Note: If you want CPU-only builds, you’d specify torch==2.12.0+cpu and add
# --extra-index-url https://download.pytorch.org/whl/cpu  
# so pip knows where to fetch the special CPU wheels. Otherwise, pip defaults to GPU builds if CUDA is available.

#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# tqdm :-
# A progress bar library. 
# We’re using it to visualize loops (e.g., training epochs, batch processing). It makes long-running tasks user-friendly.


#----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
# gunicorn :- 
# A production-grade WSGI HTTP server. 
# Flask’s built-in server is fine for local dev, but not secure or scalable. 
# Gunicorn is used when deploying our app to platforms like Heroku, Render, or Docker — it runs Flask in a robust, multi-worker environment.
